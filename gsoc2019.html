Open Source Tools - Research Group in Google Summer of Code 2019

Application Instructions

We are looking for students interested developing software for the scientific community. We need participants who are experienced in or are willing to learn about literature reviews and meta-analysis. There will be a lot of support on these topics provided by our group of experts.

Before applying, it is mandatory to read the paper that introduces to Buhos, to obtain a clear view of the objectives and features already availables

About the student proposal, we have some things in mind:

  • You'll need to have a concrete task in mind (you can pick some of the ideas listed on the page), and youÂ´ll need to define explicit success criteria.
  • We need to evaluate your interests and skills, to estimate if your skillset is in accord with your task. Therefore, please give us some information about your background and your knowledge about coding, research and literature reviews.
  • You'll need a complete schedule for your project, including specific dates and milestones.

We're accepting GSoC proposals that fits one of two categories:

  • Extensions to Buhos, with new features that improve significantly the functionality of the software.
  • Major refactoring of Buhos. As example, it would be very useful to implement the different features of the software as independent modules, available on different gems.

It is expected that all contributions provide extensive unit and integration tests and documentation. Also, as all features will be tested on its usability, is required that the schedule includes time to obtain feedback and make the corresponding changes.

The time available to develop your proposal is 4 months. A simple proposal that only takes one or two months could be rejected for a more ambitious one. The GSoC only cover coding, so other activities, like writing an article or create documentation are not suitable.

Any proposal should be sent to our Mailing group for further discussion.

Ideas

Here are some suggestions of ideas to extend Buhos. Feel free to submit any proposals for things not detailed in the list. Anyway, the ideas listed below are the most urgent needs according to our expert group, so take that into account. Also, ideas that provide both extensions to backend and frontend are prefered to another that only extends one.

  • Development of an open specification for representation of systematic reviews. This would allow users to save and upload different versions of each SLR, for backup and control version purposes. This format could eventually be used to import and export SLRs from one tool to another. As a developer, is expected that you could provide rationality of a specific documentation format, and create the modules to read and save SLR in the database.
  • Integrated federated search, allowing simultaneous searches in multiple databases. In Buhos are already implemented interfaces to Crossref, Scopus and PubMed that allow to retrieve information from external services, but is necessary to develop a search interface for users. Implementation of interfaces for other databases will be appreciated.
  • Generate canonical representations of authors and journals. Now, the authors as journals as stored as-is, but could be very useful to provide canonical representations of both object, like the canonical representation provided to documents. An user interface should be implemented, that allow analysist and administrators to remove duplicates and perform basic bibliometric analyses.
  • Improve reference search support. It could be performed directly extracting the information available in PDFs and integrating it with the information available in indexed databases.
  • Encoding of quantitative data, with descriptive statistics and meta-analysis.. The user interface for extract information should be adaptable, because one article could be report several studied, using different effect measures.
